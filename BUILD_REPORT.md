# 🛠️ Как собиралось портфолио

Кратко: что за проект, как он строился и на каком стеке. Подробные технические отчёты — в `docs/BUILD_REPORT.md` каждого репозитория.

## Как строятся проекты

Все проекты проходят единый мульти-агентный пайплайн **DEV-HUB** (идея → продукт): сбор требований → архитектура → валидация → план → сборка. На каждом стыке работают контракты-проверки, а каждое значимое решение фиксируется в `DECISIONS.md` проекта — поэтому любой выбор можно защитить на собесе.

---

## DEV-HUB V.6

- **Что это:** система оркестрации AI-агентов, которая ведёт проект от идеи до готового продукта. Это «мозг», которым построены остальные проекты портфолио.
- **Как собирался:** эволюция из версии V.5. Продуктовый слой работает в Cowork через паттерн role-switching (один агент по очереди играет разные роли), технический слой кодирования вынесен в отдельный пайплайн. Между фазами стоят контракты, которые не дают перейти дальше, пока артефакты неполны.
- **Стек подробнее:**
  - *Оркестрация* — мульти-агентные сети с явными переходами и контрольными точками
  - *Спецификации* — всё описано в Markdown с жёсткими шаблонами (ТЗ, архитектура, план)
  - *Контракты* — автоматические проверки полноты на стыках сетей
  - *Защита от обрывов* — состояние сохраняется после каждого шага агента

## Content-hub

- **Что это:** AI-инструмент авто-ведения контента в 5 соцсетях (Telegram, X, LinkedIn, Medium, Threads). Генерирует посты в моём стиле, планирует контент и собирает статистику. Существует в двух версиях.
- **Как собирался:** версия **Personal** прошла через DEV-HUB (Сети 1→2) и была собрана в Cowork; версия **Showcase** (сайт) сейчас в работе на LangGraph.
- **Стек подробнее (Personal):**
  - *4 мульти-агентные сети* — Identity Manager (онбординг личности), Planner (контент-план), Post Generator (генерация постов), Analyzer (цикл обучения на лучших постах); Backup как сквозной сервис
  - *RAG-личность* — Chroma (векторное хранилище) + локальная модель эмбеддингов `multilingual-e5-large`, чтобы посты звучали как я
  - *База данных* — SQLite в STRICT-режиме (планы → посты → версии → статистика)
  - *Чекеры активности* — Playwright для X/LinkedIn/Medium (парсинг статистики постов) + API для Telegram/Threads; с rate-limit
  - *Бэкап* — снапшоты данных в Google Drive
  - *Качество* — миграции Alembic, 40 тестов
- **Стек подробнее (Showcase):** веб-приложение на **LangGraph** (демонстрация горячего стека для рекрутёра) — в разработке.

## Learning-hub

- **Что это:** личная мульти-агентная система обучения. Агенты ведут по циклу «план → теория → конспект → карточки → практика → контроль», знания оседают в графе, повторения крутятся в Anki. Учит не ИИ — учусь я, система автоматизирует всё вокруг.
- **Как собирался:** прошёл через DEV-HUB (Сети 1+2), собран Исполнителем и развёрнут в бете; сейчас ведётся ревизия v1.1.
- **Стек подробнее:**
  - *Ядро* — Python-пакет `lh` (~9 модулей): CLI, планировщик сессий, пересборка графа
  - *Граф знаний* — Kuzu (embedded, Cypher, векторный индекс): темы, связи-пререквизиты, карточки, ошибки — как пересобираемый кэш
  - *Семантика* — эмбеддинги Qwen3 локально через llama.cpp (офлайн, бесплатно)
  - *Повторения* — Anki + FSRS: карточки уходят `.apkg`, статистика забывания возвращается в граф
  - *Карта знаний* — интерактивный граф на Cytoscape.js
  - *Надёжность* — 36 тестов, включая тест полного восстановления после уничтожения БД
